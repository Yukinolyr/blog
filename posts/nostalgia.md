##	进阶（Basic 1-10）

###	游戏的判定
和大部分音乐游戏类似，Nostalgia的音符也有对应的时间判定。如果不考虑没人用的NEAR判，那么这个游戏有◆JUST, JUST, GOOD, MISS四个判定，其中只有miss判定会中断连击，这与IIDX等游戏有所不同。各判定能获得的分数比例如下：

| 判定类型   | 得分比例 |
| :----- | :--: | 
| ◆JUST |  1.0  | 
| JUST |  0.7  | 
| GOOD |  0.5  | 
| NEAR |  0.0  | 
| MISS |  0.0  | 

不同的音符类型能获得的分数比例也有所不同：

| 音符类型   | 得分比例 |
| :----- | :--: | 
| Note（单键） |  1.0  | 
| Tenuto（长音） |  4.0  | 
| Glissando（划键） |  0.5  | 
| Trill（颤音） |  5.0  | 

可见Tenuto和Trill的分数权重还是极高的。Tenuto与Trill均没有尾判，但如果过早松开手，尾部的判定会变成good。尽可能避免在这两种音符上出现失误吧。

游戏的判定区间如下：
| 判定类型   | 区间时长（ms） |
| :----- | :--: | 
| ◆JUST |  41.67  | 
| JUST |  75  | 
| GOOD |  108.33  | 

可见游戏的最高判定区间有+-41.6ms，在音乐游戏中属于相对简单的一档了。努力以尽可能打出多的PJust为目标进行练习，会是比较不错的练习思路。

想要击打上Nostalgia的音符，只需要在合适的时间内击打note显示范围内的键就可以了。按键的宽度通常在3键到5键宽，即使没有打到正中间的键，或是同时按到了范围内的两个键也没有关系，但在练习时还是尝试尽可能击打正中间的按键吧，这对后续游玩高难度谱面会有帮助。

在进阶篇会展开讲这一点。

### 歌曲等级
首先简单介绍一下歌曲的等级系统。
这个游戏的等级系统由通常的1-12级与Real难度特有的1-3级组成。
从下图可以看到Real等级、通常等级与游戏内置定数的关系。如果不想记对应关系，可以认为$$Real n = 10 + n$$，这通常是比较准确的。

| :----- | :--: | :--: |
|1.0 | 1.0  |  |
| 2.0|  2.0 |  |
|3.0 |  3.0 |  |
|4.0 | 4.0  |  |
| 5.0| 5.0  |  |
| 6.0|  6.0 |  |
| 7.0|  7.0 |  |
| 8.0| 8.0  |  |
| 9.0| 9.0  | 1.0 |
| 10.0|  10.0 | 1.0 |
| 11.0| 11.0  | 1.0 |
| 12.0|  12.0 | 2.0 |
|12.5 |   | 2.0 |
| 13.0|   |  3.0|
| 13.5|   |  3.0|

一般而言，定数在11.0以上的歌曲属于高难歌曲，有一些11级甚至连顶尖高手也会经常出现失误。
对于刚入门的玩家而言，6-8级的歌曲通常没有复杂的节奏与配置，适合用来熟悉游戏判定与手感。对于音乐游戏基础比较扎实，或是游玩过类似游戏（如Chunithm、Deemo）的玩家，也可以直接从11-12级歌曲开始练起。

由于Nostalgia是一款非常容易地力迁移（炸鱼）的游戏，很多其他音乐游戏的技术可以直接在Nostalgia上进行使用，因此玩家在入门前期的进步速度会有非常大的个人差，甚至入坑几天收掉12下位的也不在少数，因此灵活选择适合自己的练习歌曲就显得尤为重要了。
在入坑初期，950-970k的分数可以认为是比较正常的分数，如果得分明显低/高于这个分数段，就应该考虑进行练习难度的调整了。

Nostalgia存在大量诈称/逆诈称（难度与标注等级不相符），或是个人差巨大的歌曲。因此尽可能避免以单首歌曲的分数作为参考，多打几首自己喜欢的歌曲试试吧。

###	Grade
在每首歌曲结束后，除了游戏的分数外你还会获得一个Grade分数。
<details>
<summary>Grade分数的计算方式：</summary>

$$
基础等级=Lv×难易度系数×hand系数×评级系数×play修正値
$$

$$
\text{play修正値} = \frac{85500 \times \text{判定达成度} + (10000 +  \text{总note数} ) \times \text{MaxCombo数}}{\text{总Note数} \times 120000}
$$

$$
\text{判定达成度} = \text{◆JUST数} + \text{JUST数} \times 0.3 + \text{GOOD数} \times 0.15 + \text{NEAR数} \times 0.05
$$

$$
\text{MAXGrade}  = \text{Lv} \times \text{难易度系数} \times \text{Hand系数} \times 3.3 \times \frac{95500 + \text{总Note数} }{120000}
$$

</details>

由于过程复杂，因此我们在这里不详细展开计算。但从公式中不难看出，Rank系数单独占据了一个乘区，因此满分（Pianist）与否在游戏的中后期直接决定了你的Grade高低。

$$

$$
13.0全连≈11.0 P≈131 grd
$$

$$
12.0全连≈125 grd

Total Grade是游戏中用来量化玩家水平的数值，类似Maimai的Rating、Phigros的rks值，取玩家单曲grd最高的前50首歌曲成绩加算得出。因为游戏的Grd算法非线性且P权重过高，同一难度下不同歌曲的难度差距较大，因此实际上Total Grade并不能准确地量化玩家的游戏水平。将其作为一个简单的参考，而非游戏的终极目标，能让你更好的享受游戏本身。~~你打多高也没人在意~~

### 合适的手型、发力
与钢琴类似，游玩前让手臂、手腕与手指完全放松，手指自然弯曲呈弧形，类似握住小球的形状。将此手型放在键盘上，即可达到最佳位置。手指应以第一关节为“支撑点”，不可塌陷或凹陷。过于弯曲或伸直的手指都会导致紧张和控制力下降。

与钢琴有所不同的是，Nostalgia的按键手感较为松软，因此想在键盘上保持手型也是相对困难的一件事情，对手指的掌控力与力量都有所要求，一开始做不到也很正常。让手指慢慢适应按键的手感，逐渐找到适合自己的手型最为重要。为了保持手型而让手腕僵直，或是影响到手指的灵活性是不可取的。

按键时应使用指尖的下侧，保持手指弧度。因此，将指甲修剪到合适的长度或许是必要的（就像游玩其他键盘类音游一样）。大拇指自然伸直，使用侧边压键，更灵活自然。当然，在一些特殊情况下手型与触键的方法也要有所调整，例如在处理一些多押谱面时，将手张开增大触键面积，也是提高容错的一个好方法，当然这只适用于Basic模式。

###	该怎么练习
在入门阶段，笔者认为最重要的就是三件事：找到判定、找到定位、学会使用拇指。

找到判定与定位都很好理解，尤其像Nostalgia这样一个有大量手部位移的游戏，想把把每个键都精准的定位，这样的映射是需要一段时间去建立的。特别对于初次接触街机音游的玩家而言，相比起移动端音游“所见即所打”，Nostalgia的画面与游戏按键是分离开来的，建立从视觉输入到手部动作输出的映射就会更加困难一些。参考指南2.2的内容，选择自己喜欢的歌曲多加练习吧。

学会使用拇指是每一个Nostalgia玩家的必修课。作为我们五根指头中最强壮、也是最灵活的手指，我们在游玩大部分音游的时候却总是会忽视掉它（很多时候是因为游戏设计）。但在Nostalgia中，对拇指的使用绝对是重中之重，可以说一个人即使不使用4、5指也能成为一个优秀的Nostalgia玩家，但如果不使用拇指，那连入门都会变得十分困难。尝试在游玩时多练习拇指的准确定位，拇指与2、3指的连携动作，如1212、1313，这对于后续的进步是十分有益的。

作为一个玩法模拟钢琴演奏的游戏，我们玩Nostalgia的最终目标一定是灵活的使用十根手指进行操作。很多玩家一开始或许抱有畏难心理，觉得想成为“十指琴魔”实在是太难了。那么不妨就从最强壮的拇指开始循序渐进的练习，通过不断地建立新突触与提升手指肌腱力量，你会发现开发手指并没有那么难。加油试试看！

## 进阶（basic 10+）

### 按键机械结构
若想进一步提升自己的游戏水平，我们就需要先对游戏本身有更加深入的了解，无论是软件还是硬件。Nostalgia的按键结构与大部分街机音游都有所不同，了解按键结构可以让我们更好的设计合理的发力方式与手法。
![拆卸图1](/images/nostalgia_introduction/3.1.1.jpg)
![拆卸图2](/images/nostalgia_introduction/3.1.2.jpg)
将屏幕下侧的IO BOX进行如上图的拆卸，我们就能看到框体内部的按键机械结构了。
每个按键被两侧贴着黑色泡棉的挡板和下侧的卡扣限制，一上一下使其被固定住，有效避免了按键脱落，或是从一边翘起来的问题。

![拆卸图3](/images/nostalgia_introduction/3.1.3.jpg)
贴着1mm泡棉的挡板。像这样的挡板有两个，一前一后配合卡扣限制按键的移动。

![拆卸图4](/images/nostalgia_introduction/3.1.4.jpg)
![拆卸图5](/images/nostalgia_introduction/3.1.5.jpg)

拆除挡板，并将其中一颗按键从卡扣上拆下来，可以看到每个按键下方有两个50g弹簧，主要负责按键的回弹。
每个按键下面还有两条厚度为1mm的黑色泡棉，在限制按键位移高度的同时提供了一定的缓震与回弹，优化按键手感。两条黑色泡棉中间是按键的感应传感器。

![拆卸图6](/images/nostalgia_introduction/3.1.6.png)
图片@i. 
将按键从卡扣上取下，我们能看到每个按键的底侧都有一个圆形的磁石，游戏通过每个按键下方对应的传感器去感应磁石的下落，并由此来判定你是否按下了这个按键。一般来讲，只要按键上的磁石达到键程一半左右的位置，下方的传感器就会将其认定为一次信号的触发。这与大部分通过微动开关进行按键判定的街机游戏有所不同。

传感器感应到磁石落下的速度越快，则按键落下的速度越快，说明你触键的力量越大，反之亦然。Recital模式的轻重判定便是以这个方式对不同的触键力度进行识别的。

得益于良心的卡扣&挡板设计，理论上在一个按键的任何位置对其进行敲击，传感器都能感应到磁石的下落，而不必担心按键的另一侧因翘起来而失去判定。但在实际的游玩中，4/5指这样触键力量较为薄弱的手指，很可能并不能将一个按键完全按下，那么如果这时触键的位置偏离中心的磁石位置，就有一定可能导致这一次按键不被识别。因此，在日常的练习中，努力将触键的位置调整在按键的正中间，使按键能够整体垂直地下落，是避免按键吃音的通解。

>按键的几个挡板通常是Nostalgia卡键的罪魁祸首。如果在游玩过程中出现按键卡在底部不回弹的情况，多半是卡扣挡板处的泡棉融化粘住按键所致。（笔者在日本的七个街机厅进行游玩，均出现了类似的问题）

按键左右两侧的白色挡板可以调整按键的松紧程度，如果夹得过紧，两侧的按键会出现卡键的情况，反之则会让按键过于松弛，影响游戏手感。在出现机况问题时，尝试与街机厅的工作人员进行沟通，检查一下按键几个挡板的情况吧。

![拆卸图7](/images/nostalgia_introduction/3.1.7.jpg)
间隔过于紧的右挡板。可见最右的按键被挡板卡住了。

### 判定保护
很多游玩过中二节奏的玩家们应该都了解判定保护这一概念。这是一种专为中二节奏设计的特殊的判定触发机制，设计初衷是为了避免冗余的按键触发，进而优化游戏的手感。而这种独特的判定机制，也使得玩家以此为基础设计出许多手法，大幅度提升了游戏的可玩性。

或许有必要解释一下什么是冗余的按键触发。

- 今ぞ♡崇め奉れ☆オマエらよ！！～姫の秘メタル渇望～ MASTER谱面
![实例1](/images/nostalgia_introduction/3.2.1.png)


可以看到，谱面第83到90小节都有这样一连串的四格纵连配置。如果以不刻意控制手型的方式进行游玩，我们在触碰游戏的触控板进行按键时，会出现多个手指同时蹭到不同按键的情况，也就是“摸了一下被判定成好多下”，进而误触后面的按键，影响游戏手感。

判定保护有效地解决了这个问题。中二节奏的判定保护，是通过缩小重叠按键之间的判定区间实现的。按键之间的判定区间被缩减，在按键时多余蹭到的手指就不能判定到后面的按键，手指的误触输入也就被无效化了。

那么Nostalgia作为一款设计类似的无轨音游，自然也具备类似的机制来达到这种判定保护的效果，但实现的方法却有所不同。为了更好的解释这一较为复杂的机制，我们需要进行一些定义与性质的补充。

- [nostalgia的判定保护机制](https://pub.yukino.io/nostalgia_judgment)

简单来讲，如果你的按键输入没有按到音符对应的判定区域，而是按歪到判定区域的左/右侧三个键，则会触发一种惩罚式的判定——Near判定，你所击打的这个音符像被正常判定了一样消失，但是你无法获得任何分数。

在实际游玩中，我们通常是关闭Near判定的，但这并不意味着Near的判定机制被彻底关闭，实际在游玩中，内置的Near判定依然会对每个输入产生影响。设置中的“关闭Near判定”，指的是“让能够使音符判定到Near判定的所有输入失效”，也就是说，所有在一定程度上按歪了的输入，都会因为这个设置而失效，进而避免了判定区域外的冗余输入触发本不该被触发的音符。而对于判定区域内的输入，当有一个输入成功判定了音符，则该音符判定区内所有同一帧的输入会全部失效，这也就避免了判定区域内的冗余输入触发不该被触发的音符。

如此，我们便实现了对冗余输入的全部处理。需要注意的是，对冗余输入的处理仅对音符的判定区域内有效，超出判定区域的冗余输入仍然会导致误触的发生。因此，对每根手指定位的精确控制仍然是十分必要的。

### 音阶处理——连打与刮奏

#### 连打 
对于刚入门Nostalgia的玩家而言，连打与刮奏应该是学习成本最低的手法了，因为它通常不需要考虑多根手指之间的协调，也没有什么额外的技术，只需要一根手指进行操作。

连打处理适用于较慢速的，窄间隔的音阶（后文以楼梯代称），只需要使用一根手指，在合适的时间进行对应的位移与击键操作（戳）即可。每个人擅长的连打发力方式不同，因此本文在此不多做赘述。

连打的短板在于，一旦楼梯的间隔稍宽或速度稍快，连打处理的难度就会以指数倍增长，因此很考验玩家的身体机能与其他音游携带的地力。一些玩家或许能轻松地进行160+bpm16分的位移连打，而像笔者则很难做到这一点。因此，对于高速楼梯是否还要进行连打处理，就因这种极大个人差的存在而见仁见智了。

总而言之，在遇到低速且复杂的楼梯时，如果使用连打处理，相对来说就不需要考虑对应的运指顺序，谱面的处理难度也会随之骤减。

- [一个很好的戳楼梯实例——诸葛暗](https://www.bilibili.com/video/BV1nCxfzEEgD/)

#### 刮奏
刮奏处理适用于快速的，间隔较宽的楼梯，只需要使用一根手指，在合适的时间进行类似slide的滑键操作即可。滑键的缺点是分数稳定性差，尤其在慢速窄间隔或不等间隔的楼梯里，经常会出现低分判定，甚至miss。

因为楼梯之间音符与音符的横向距离如果过窄，根据3.2中所说的机制，同一帧数内可判定音符的Unear覆盖区域就变得越小，我们就越有可能在刮奏时提前刮到note的Near区域，导致一连串坏判定的触发。但对于极快或极宽的楼梯（如Piano体操第一，turn the story），正常的运指方法显然很难处理，这种时候滑键就是正解。

- [一个很好的刮楼梯实例——Fork512Hz](https://www.bilibili.com/video/BV1rkxLzFEu3/)

连打与刮奏都不是Nostalgia这款游戏的主流手法，但实际上由于运指法的高学习成本与不稳定性，连打与刮奏在游戏之中的使用率是相当高的。如果在使用这两种手法时能比一般的运指法更稳定的获得分数，那么我推荐你去使用这两种非常规手法。

当然，过于依赖非常规手法会导致玩家缺乏对于一般运指技能的练习。假设你可以比较轻松的使用连打处理140bpm的16分楼梯，却不能用运指法处理同等，甚至更低速的楼梯，那么在之后需要处理150bpm的16分楼梯时就难免会陷入瓶颈。因为连打处理的能力提升相当缓慢，为了打上更高速的楼梯，再从低速的楼梯开始从头练习运指就有点晚了。

水之积也不厚，则其负大舟也无力。为了避免这种情况发生，在日常的练习时，哪怕遇到了在你连打处理能力范围内的配置，也请使用运指法来进行练习吧。

### 音阶处理——基础运指
直到Op.2，在绝大多数expert 12级的谱面中，都不会出现超过7连的楼梯键型。即使像莫扎特K545，夏三这样几乎全部以简单朴素的一键宽楼梯组成的曲目，也被玩家们视为一生之敌。谱面中一旦出现楼梯，这首歌曲的玩家平均分数就会直线下滑，由此可见楼梯配置的处理难度。

为了更好的处理这些楼梯配置，我们首先要从最简单的短楼梯配置开始分析。

这里有一个一键宽5连的楼梯配置，是nos中最基础的短楼梯配置。在看到这样一个配置出现的时候，我们应该如何处理呢？最标准的打法当然是使用拇指–食指–中指–无名指–小指来依次敲击。

在中文的Nostalgia讨论圈中，我们通常使用一套比较简洁的语言去表示这样的一组，或是几组运指法：数字12345分别代表双手的大拇指/食指/中指/无名指/小指，于是我们就可以以一串数字标明每个手指触键的顺序，而括号（）代表括号内的数字是需要不同的手指同时触键，也就是在同一时间进行多押的。

像上面这串楼梯的标准运指法，我们就可以用【12345】去表示。

不过相信绝大多数不能处理好这种配置的玩家，并不是不知道这串楼梯是这样打，但他们还是很难打上。那么问题究竟出在哪里呢？

#### 根本没有按够5下
对于一个5连的楼梯配置，如果你根本就没有按够5下，那自然就会出现miss。这是显而易见的道理，但至少有80%的短楼梯配置失误都是因为这一点，这很大程度上是Nostalgia的判定保护机制与按键手感所致。

当我们在一个全/半配重的真实钢琴键盘上进行演奏时，由于琴键本身存在的配重，我们的手指是需要给予一定的发力才能够将琴键按下去，进而击弦发出声音的。换句话来讲，琴键的配重使你手指触发声音的力度阈值变高，只有非常明确的击键行为才能使钢琴发出声音，这种明确实际上就是控制力的一种体现。

熟练的钢琴演奏者能够通过触键技巧做到细微的音色控制与强弱变化，也建立在这一点上——如果钢琴的琴键轻盈到一碰就能击弦发出声音，那演奏者又该如何去控制音色与强弱呢？

但Nostalgia的键盘却完全不是这样。参考3.1，每个按键只由两侧的50g弹簧支撑，手指仅仅需要微微发力，甚至蹭一下，就能将按键按到底部。按键底部类霍尔开关的设计使按键的触发变得异常灵敏，玩家在按键时只需要按到键程一半的位置就能触发一次输入。

触发输入的力度阈值变低，这虽然使得玩家可以轻而易举地触发每一个按键，但也意味着玩家对于输入这一行为的控制力变低了。玩家需要更加细致的控制自己的肌腱，通过微操去进行每一次的触键，若是稍有不慎就会出现“不知道自己在打什么”的情况。

回到我们先前的例子。对于一个5连的楼梯配置，我们的通常解法是12345，其中大拇指/食指/中指对于我们而言是相对强壮灵活的手指，因此我们可以比较清晰地打上前面的123。（你可以在桌子上，或是街机按键上尝试一下轮流抬起123指，并把它放下，这不是一件困难的事情）但对于后面的45，却有可能完全不是这么一回事。（你可以再尝试一下轮流抬起345指）这是为什么呢？

![3.4.1](/images/nostalgia_introduction/3.4.1.jpg)

这是一张手部伸肌的功能解剖图。当你按下琴键后，需要让手指回到初始位置，准备下一次按键的动作，这个 “抬指” 动作主要由伸肌腱驱动。

我们重点关注图中腱间结合的部分。什么是腱间结合？你可以把一只手的手背朝上平放在桌子上。抬起中指，可以看到手背中线处有腱突起。保持中指抬起，用另一只手摸摸中指与无名指之间，你就会感受到一条横向的筋膜组织略微隆起，这就是其中一个腱间结合。

有五根像“绳子”一样的伸肌腱并排经过手背，但在他们之间用几条“橡皮筋“系在一起。当你试图拉动当你试图单独拉动其中一根（例如无名指那根），橡皮筋会把一部分传给旁边的绳子（中指、小指），于是它们也会跟着动，或者反过来牵制那根，其中4指与5指之间的牵制力是最强的。这就是绝大多数人的无名指独立性更差的原因，因为无名指受牵制的影响本来就很难独立抬起。

由于手部生理结构的这种特性，我们在处理12345当中的45时，由于伸肌无法独立抬起无名指，我们的运指就会受到一定的影响。

可能有读者会想到，既然我们很难控制伸肌去独立抬起无名指，那么为什么不先忽视“抬起来”这件事，先去研究如何将按键“按下去”？

事实上早在18-19世纪，由于浪漫主义时期钢琴技术的发展越来越追求十指独立，快速均匀，许多钢琴家与学者就已经对如何“抬起来，按下去”展开了研究。像作曲家舒曼，就曾因使用固定无名指进行手指练习的装置而落下终生残疾（也有一说是舒曼试图自己切开4/5指的腱间结合），更早的克莱蒙蒂也曾在开发钢琴乐器时制造了一系列的手指练习装置。

在20世纪，由于外科手术技术的发展，肌腱分离手术的实现成为了可能，有许多钢琴演奏者进行了这种切断的手术，但效果却并不理想——手术并不能显著提升手指的独立性，并且使手指的力量大幅下降，稳定性变差。这主要是由于4指和5指主要由尺神经的分支部分控制，而2、3指主要受正中神经支配。这意味着即使将肌腱分离开来，对于从大脑到肌肉的神经控制信号，4指和5指的“线路”也有部分重叠，因此难以完全独立分化。

![3.4.2](/images/nostalgia_introduction/3.4.2.jpg)

在这样的背景下，现代钢琴教育（尤其俄罗斯学派、现代欧洲体系）已经逐步放弃这种“机械式独立指”的理念，将对手指技术的关注从如何“抬起来”转移到如何“按下去”，并强调整体手臂的重量控制与协调运动链，也就是手腕、前臂、肩的联动。在现代钢琴教育中常提到的“手臂/腕放松，用指尖去带动整体发力”就是此类理念的一个缩影。

一些学派甚至指出：手指，尤其是环指（ring finger，即无名指）抬起的高度不必高于手本身的平面，按键的力量应该更多依靠整体的协调，而非手指本身。

然而对于Nostalgia而言，想做到这样的协调绝非易事。前文中我们提到，Nos键盘的触发阈值相对较低，玩家本就难以控制每一次输入，倘若再加上手腕、前臂、肩的联动，只会让我们对手部输入的控制变得愈发困难。

此外，Nos键盘的按键键程（特别是触发键程）也显著地短于钢琴琴键，倘若再降低手指抬起的高度，手指在落键时用来“助跑”的距离也就变得更短了，我们的发力也进一步受到了限制。因此，对于Nos而言，“抬起来”并不是一个可以被忽视的事情，我们仍然需要去和自己手部的生理结构作斗争，通过拉长手指击键的距离来实现“颗粒分明”地运指————这种技术上的矛盾，导致我们很难在游戏操作中精准地使用4/5指处理谱面。

</details>



在这样的困难下，玩家在处理12345的配置时，很容易将后面的45打成（45）/4/5 。对于4/5我们比较好理解，无非是在运指时失去了对无名指或小指的控制，没有将其按下去。但将45打成（45），是很多人都没有意识到的问题，也是绝大多数楼梯失误的成因。

3.2附录中提到，当有一个输入成功判定了音符，则该音符判定区内所有同一帧的输入会全部失效（满足附录7.2性质的音符组合除外），于是在同一音符判定区内输入的（45）判定到第四个按键后失效，第五个按键就这样被漏掉了。

由于Nos键盘的低触发阈值，玩家通常很难在触键时得到准确反馈——很可能你认为自己清晰而准确地依次按下了12345，但实际上在你抬起并按下无名指时，小指或许就不小心因为代偿同时蹭到了按键，这会导致实际的输入变成了123（45），但在不观看手元录像的情况下，你很难发现这一点。

那么我们应该如何解决“根本没有按够5下”的问题呢？很遗憾，立竿见影的的解决方案暂时不存在。但我们仍然有间接的解决方案：
- 不使用4/5指
- 锻炼4/5指

**不使用4/5指**：这是非常现实，成功率也很高的方法。例如对于一个5键楼梯的配置，我们可以使用12123，或是12312的转指手法来处理。（有关转指的手法与技巧，在下文3.5中会予以详细的说明）但由于拇指通常比较粗，容易同时压到多个键，也就容易出现将1打成（12）的情况。（下文定位中会予以说明）

此外，转指作为一个需要手臂/手腕/手指多方面协调的技术，我们在高频率使用时也会出现定位不准，或是来不及操作的情况（手指卡住了没转过去），导致楼梯掉一整串。

**锻炼4/5指**：如上文所言，想要要、训练那些几乎一辈子都没怎么用过力，又在生理结构上存在限制的手指，比想象中要困难得多。但适当的练习，仍然能有效提高手指机能，尤其是在早期阶段。很多玩家为了玩好Nostalgia去学钢琴，这是一件效率不高的事情，但像一些钢琴中的基本练习（如Hanon 1-20）确实能高效的锻炼手指。

比起练钢琴，我认为在平时无论玩什么谱面，都刻意去练习多样化的手指发力，使用上薄弱的4/5指才是更为重要的。毕竟玩Nos本身才是提高Nos水平效率最高的方式。当然还有很多练习4/5指的方法，在后文的练习策略中会详细展开这一点。

此外，并不只有4/5指会存在没按够5下的问题。哪怕是使用我们比较擅长的123，在处理高速谱面时，也会出现类似的情况。所以在处理楼梯时，请对这些细节多加注意吧。

#### 定位出现了问题
有时候即使我们确实按够了5下，但楼梯依然会华丽丽的掉一整串，这是因为没有击中正确位置的音符。

从第一个音开始就按错的情况很少发生，我们运指的定位问题通常发生在第二个音之后：
- 按得太窄
- 按得太宽

**按得太窄**：比如你的大拇指同时按到了A/B两个键的缝隙，导致A/B键同时被按下，但食指在B键的判定还没有完全释放（没有足够抬起）之前又按了一次B键。这会导致第二次的B键根本不会被判定，那么后续的整个楼梯就一路崩掉了。

**按得太宽**：明明12345对应ABCDE键，却因为手指过分张开打成了ACDEG。这样最后一个音符就超出了按键的判定区域，必然Miss。

好在Nostalgia对判定区域要求不严格，向外按宽一格也能被判定到。即对于一个基础5连楼梯，5次击键里可以允许出现一次击键失误（类似13456）。因此按得太宽导致崩盘的情况，一般会在连续的多楼梯/长楼梯配置中出现。

在基础的5连音阶中，主要的定位问题还是集中在「按得太窄」上。因此我们在此也主要解析「按得太窄」的解决方案：
- 张开一点
- 练习定位
- 学会断奏

**学会断奏**：如果在击打每一个音符时都能做到「快速按下并立即放开」，那么即使出现定位问题导致A/B键被同时按下，B键也会因为立刻被放开而不影响第二次B键的判定（类似于处理SDVX/Maimai纯外的纵连）。

当然，这并不是一件容易的事情，通常需要借助其他音乐游戏的地力与游玩经验。依笔者浅薄的经验来看，断奏技术的关键在于协调和对按键键程的把握。

对于协调，我们要明确「前一次按键的松开」和「后一次按键的按下」实际上并不是独立的事件，我们需要将其视作一个整体，即在前一次按键松开的过程中，我们就要开始对后一次按键的按下进行操作了。

如果这个衔接过于快，则会使第二次击键在第一次的判定还没有完全释放前完成，如果衔接过于慢，后一次击键的时间过程就会被压缩，对击键的时机和发力产生影响。保持这样的协调需要相当高的熟练度，特别在对高速的断奏进行处理时，因此在游玩时请多多注意这一点吧。

按键键程的把握，实际上就是通过控制缩短击键的键程，使前一次按键松开的过程变短，这样留给第二次击键的时间就变得相对充裕了。

Nos的类霍尔触发使我们不需要将按键按到最底，只要保证按键刚好下落到能够触发按键的位置，就可以将其立即松开。当然，想做到这一点也需要相当的熟练度，倘若下键的位置过于浅导致判定根本没有被触发，那这样的操作就得不偿失了。

笔者事实上相当不擅长任何音乐游戏中与此相关的技术，因此没办法总结出一个完整的练习方法论，只能给出一些个人的偏向于经验主义的理解。如果你在这方面有自己的见解，无视这部分内容即可。



**张开一点**：前文中提到，对于一个基础5连楼梯，5次击键里可以允许出现一次击键失误，也就是说按的宽一些可以获得更多的容错。因此如果在处理普通5连，甚至4连音阶时总是因为把控不好定位而出现漏键，那么有意识的按宽一点确实是一种解决方案。

一个很好的例子就是Etude op.10-4激流，红谱的右手重复4连，紫谱的左手重复4连都很容易因为机械性反复导致的疲惫而出现手指的定位问题，此时将手张大一点就能很好的提升容错。

**练习定位**：上面的解决方案终究是治标不治本。在游玩时养成精准定位，每根手指都按到音符中间的习惯才能解决根本问题。对于手指的定位练习，首先要刻意练习手指在每一次击键时都要按在按键，而非琴缝上。

通过这样的练习，每个手指都会养成相对的肌肉记忆，即当你的任何一个手指按到按键上时，你就能知道其他手指应该所处的正确位置，就像相对音感一样。在掌握了这一点之后，再去建立每个按键与屏幕音符之间的映射，就显得事半功倍了。

笔者目前还处在建立相对记忆到建立映射之间的过渡阶段，一起努力练习吧。

总之，玩家无法处理一键宽5连音阶的原因，大体不会脱离以上两点。

当然，Nostalgia的另一大难点在于两只手共同的协调，单纯的楼梯倘若伴随另一只手的不同配置与节奏型，我们就会顾此失彼，原本简单的配置也会变得困难。因此，找到一套适合自己的方法应对这种5连楼梯，并通过练习将其内化到自己的肌肉记忆，达到不经过大脑就能平A出来的水平，那么协调的难度就会大大降低。

我们通常称这种不经过大脑处理，可以运用自如的运指策略为「公式运指」。

### 音阶处理——公式运指与转指

所谓公式运指，最核心的意义并不在于它有多“正确”，而在于它能够被稳定地复现。

对于楼梯这种看似种类繁多、实则高度相似的配置而言，如果每一次都要在读谱后临场思考“这一串该怎么打”，那大脑的处理速度往往是跟不上手的。尤其是在另一只手还有配置、或是楼梯本身速度较快的时候，哪怕只是短暂地犹豫一下，后续整串都很容易一起崩掉。

因此，比起执着于对每一种楼梯设计一种最优解，公式运指更像是一种将常见配置压缩、归类、再内化的过程。它的优点有两个。
- 大多数楼梯都可以尽量交给更强、更稳定的 1/2/3 指来完成，减少对 4/5 指的依赖。
- 许多不同长度、不同形状的楼梯，最终都可以拆解成少数几种相似的基本型。这样一来，不同楼梯之间的学习成本就会显著下降，稳定性也会更高。毕竟你总是在用相近的方式处理相近的问题。

当然，这并不意味着 4/5 指就不重要。如上文所述，它们一定是要练的。只是对大多数玩家而言，前期如果一上来就执着于用 4/5 指把一切都正确地弹出来，往往只会产生挫败感。

相比之下，先建立一套更容易上手、更容易稳定复现的公式运指，再在平时的练习中逐步补强 4/5 指的机能，通常会是更现实的练习路线。

在讨论公式运指的不同楼梯型之前，我们需要先学习公式运指所需要的演奏技巧——转指。

#### 转指的方法与技巧
转指是指在弹奏过程中，一个手指从另一个已经按下键的手指下方穿过/上方跨过，以到达需要弹奏的下一个音的位置。通过这样的方式，我们就能使用不相邻的手指去击打连续的按键，并将手位线性调整到想要的位置了。在nos中，绝大部分转指的操作都建立在拇指上。

例如，对于一个上行的5连楼梯，我们可以在拇-食-中按完前三个按键后，不使用相邻的4指去打第四个键，而是把拇指从中指的下侧穿过去，使其能够按到第四个键。下行的5连楼梯同理，假设我们用中-食-拇的的顺序按完前三个按键之后，可以把中/食指跨过正在按键的拇指，使其能够按到拇指左侧的第四个键。

若只从字面理解，很容易把转指想成“把拇指硬塞到手掌下面”；但这种理解往往会把动作做大，导致节奏发紧，甚至在换位处出现明显卡顿。实际上，转指本质上并不是某一根手指的突击，而是一次手位的自然交接。手臂/手掌/拇指的协调配合是极其重要的。

在上行楼梯中，当前一个手指按下时，拇指其实就应该已经开始朝下一个目标键靠近；下行时也是同理，新的手位应当在旧手位结束之前就开始准备。

钢琴教学里常强调，拇指不要等到最后一刻才“补位”，而应与手部、前臂的微调同时进行。这样做的好处，是可以把换位处的大动作拆掉，让整串楼梯更均匀，也更容易维持节奏。

为了避免出现上述的问题，在转指时可以注意几个要点：
- 拇指的接触点尽量落在指甲与皮肤交界附近的角部，而不是过多使用侧面去“蹭”键。
- 拇指保持轻微弯曲，不要完全打直。
- 手型不要因为转指而塌掉，手掌内部仍应保留基本支撑。
- 手腕可以有很轻微的配合，但不应通过明显下砸或外甩来“制造”转指。

归根结底，一个好的转指动作看起来往往并不夸张，甚至会让人觉得只是手自然地滑到了下一组位置上。

郎朗在他的钢琴大师课上曾分享过一个技巧：拇指在转指过程中就像撑杆跳的那个杆，你可以借助拇指下按撑起的力量，将整个手掌顺势带到你所想要的手位。这个技巧在nos中依然适用，且对于高速的1234型大跨度楼梯有奇效。

#### 上下行楼梯的基本型

公式运指的核心，是通过转指的技巧，让更强的手指重新接管后续的音，从而把一整串楼梯拆成若干个自己熟悉的基本型。

对于上行楼梯而言，最常见的基本型是 `123` 循环。例如一个 5 连上行楼梯，可以拆成 `12312` 或 `12123`，；一个 6 连上行，可以直接处理为 `123123`；一个 9 连上行，则可以进一步写成 `123123123`。

也就是说，许多看起来不同的上行楼梯，本质上都只是同一种手法的延长与重复。

下行楼梯同理。其核心通常是 `321` 循环。一个较长的下行楼梯，很多时候都可以先按 `321321321` 这样的思路去理解。

除此之外，`1234` 循环也是非常重要的一种基本型。虽然它会用到 4 指，但在实际游玩中依然时常使用。原因并不复杂：Nos中的许多歌曲都建立在较清晰的四拍子结构上，`1234` 这种循环和每一拍、每一小节的分割天然更容易对齐，因此在处理较长的音阶时，它不但方便找节奏，也更方便重新定位。

很多时候，长楼梯看似复杂，实际上只是若干个 `1234` 或 `4321` 的连续而已。

#### 短楼梯

在本文中，我将 5 连及以下的楼梯视作短楼梯。

这类配置的特点在于，它们从长度上来讲，并没有长到必须依赖连续的转指去处理。也正因如此，短楼梯往往是建立公式运指观念的起点。

例如最基础的 5 连，上行可以写作 `12345`，也可以拆成 `12312` 或 `12123`；下行同理。

对于大多数玩家而言，转指通常会是更稳定的通用解，因为它减少了 4/5 指直接承担主体工作的频率，练习的成本也相对更低。

但如上文所述，这并不意味着 45 指可以被完全回避。因为当速度进一步提高，或是楼梯与其他配置发生复合时，继续一味依赖转指，对拇指和手位换位的负担反而会越来越重，此时 4/5 指的参与就会逐渐变得必要起来。

换句话说，公式运指，特别是短楼梯公式运指并不是唯一的标准答案，而是学习成本相对较低，能够稳定复现的基本手法。尽可能把基础5连的公式运指练到建立全映射的状态——看到配置时不必再临场思考。

#### 长楼梯

从 6 连开始，楼梯就进入了另一个阶段。因为这时配置已经不可能只靠五根手指一口气处理完，必须开始依赖循环与拆解。也正因如此，我会把这类配置统称为长楼梯。

长楼梯看起来很长，但对于大量常见的 16 分楼梯而言，它们往往都可以先按拍拆成几个更小的部分来看。也就是说，比起把整串当成一个连续不断的巨型配置，不如把它理解成几个彼此相连的小楼梯，而在每一小段的起点，都重新完成一次定位。

例如一串上行的长楼梯，很多时候既可以按 `1234 / 1234` 来打，也可以按`123 / 123 / 123`这样的方式去理解。这里真正重要的，并不是你是否和别人写出完全一样的数字，而是你能否明确地意识到：前一个小段结束之后，下一小段其实已经是新的起手，你需要通过各种方法去调整自己的节奏和手指定位，维持一个好的手部姿态去处理下一段小楼梯。

如3.4所述，在处理简单的5连楼梯时我们习惯把手指稍微张开一些，因为判定保护对非重叠的击键较为宽容。然而在连续的多楼梯/长楼梯配置中，楼梯按得太宽可能导致连锁反应————每一次楼梯倘若都按宽一个键，那么在第二次循环进行时，最后一个按宽的键就会脱离判定区，导致一整串楼梯崩盘。由此可见，在每一个小循环启动前调整击键的定位和节奏是尤其重要的。

当然，这种处理方式也不是万能的。速度较慢时，这种重新定位的过程往往会比较自然；但速度进一步提高后，这种“每一段都明显地断开、重新起手”的做法就会逐渐变得困难。

到了这种时候，通常有两种思路。其一，是依然保留分段的意识，只是将中间的几个音略微前置，尽量给下一段的起手腾出空间（虽然这很不音游）；其二，则是直接修改尾部或连接处的手法，让相邻两段之间的衔接更顺。

前者更强调节奏与颗粒感，后者更强调连贯与手感，实战中两种方法都会用到。后者在复合/往返楼梯中更常使用，后文中会详细说明这一点。

细心的朋友应该已经发现，我在前面提到长楼梯可以按 `1234 / 1234` 来打。这不仅意味着长楼梯可以使用1234型，还意味着我们在上行楼梯结束时，需要考虑是否将4/5指加入进来。

举个例子：对于一个7连的上行，我们在初见的时候通常是按照最熟悉的`123` 型来打，那么我们就会打成`1231231`，进行三次转指。

你很显然发现了这个指法的不合理之处：我们为了打最后的一个键，多进行了一次转指，对这个键付出的成本太高了。正常的解法是在第二个循环后直接加入一个`4`，即`1231234`。

在上行长楼梯中，如果每次转指后都要思考要不要加4/5指实在过于复杂的一件事，而公式运指可以将楼梯拆成类似于`xxx xxx xxx｜xxxx`的型，我们只需要考虑楼梯的最后几个键该如何收尾即可。这也是公式运指的一大好处。

对于下行长楼梯，由于中/拇指跨过拇指的操作相比上行拇指穿过中指的操作要简单很多，所以我一般不考虑在`xxx xxx xxx｜xxxx`的最后打成 `4321` ，毕竟从`321`到`4321`的转变太容易失误。

但对于手位靠右侧的长楼梯，很多时候我会考虑打成`xxxxx｜xxx xxx`的型，即楼梯开头用`54321`起，再接`321`循环。因为手指内滚的`54321`往往比`12345`更顺手（你可以试试），不会给手带来太大的压力。

还有重要的一点：`54321`或`4321`起意味着手移动的距离变小了。如夏三的开头，右手是重复的下行长楼梯，且每个楼梯之间的时间间隔极其短。如果使用`321`起，很可能会出现手部位移不到位，从第一个键开始就歪的情况。`54321`起能减少右手两个键位的移动距离，对于这种时间间隔短的楼梯而言，可以说提供了相当高的容错。

#### 小结

归根结底，公式运指并不是去死记某一套唯一正确的数字，而是让你在面对楼梯时，能够迅速从几套熟悉的基本型里调出最稳的一套。对于绝大多数玩家而言，这种“先把楼梯化繁为简，再把简化后的处理方式反复练熟”的过程，往往比执着于每一种配置都追求最优手法，更能真正提升实战中的处理力。

当你已经能用这种思路去理解楼梯时，后面那些看起来越来越长、越来越快、越来越怪的配置，也就不再是完全陌生的东西了。很多时候，它们只是同一个问题换了一个长度，换了一个速度，或是换了一种更麻烦的样子而已。


### 复杂的运指技巧、注意事项与实例

#### 宽楼梯

宽楼梯相对于前文的一键宽窄楼梯，一般指二键宽的大跨度楼梯。

然而宽楼梯和窄楼梯并不是两套完全不同的体系。很多时候，它们在公式运指上的基本思路其实是一致的：该循环的时候循环，该转指的时候转指，该让 4/5 指收尾的时候也一样要让它们进来。

真正的区别在于，宽楼梯对于手部横向移动的要求会明显更高。尤其是在使用转指处理时，本来就已经增加了部分手指的往返距离；若楼梯本身还比较宽，那么即便理论上的运指公式没有变，实战中的难度也会显著提升。因此，宽楼梯虽然仍然可以套用和窄楼梯相同的运指框架，但往往更考验玩家能否在移动中依然维持住原有的定位精度与节奏感。

也正因为如此，宽楼梯通常会比窄楼梯更早暴露出“公式是知道的，但手跟不上”的问题。这并不意味着公式本身失效了，只是意味着你需要在更高的速度、更大的移动量之下，把同样的公式继续维持住而已。

#### 关于左右手

左右手在楼梯处理的逻辑上，并没有本质区别。

右手能用的公式，左手原则上也能用；上行、下行、短楼梯、长楼梯、宽楼梯，对左右手而言也都遵循相同的思路。

之所以很多玩家会觉得左手楼梯明显更难，更多还是因为左手本身通常更弱，而 Nos 的左手楼梯配置又相对较少，导致练习机会不如右手那么充足。说到底，这里并不存在什么特殊的“左手理论”，更多只是熟练度与机能上的差距而已。
